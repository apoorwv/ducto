# Expression Reference

ducto uses a safe expression language for pricing formulas. The same syntax works identically in Python and JavaScript.

## Arithmetic

| Operator | Example | Description |
|----------|---------|-------------|
| `+` | `input_tokens * 0.01 + output_tokens * 0.03` | Addition |
| `-` | `-cache_read_tokens * 0.001` | Subtraction / negation |
| `*` | `input_tokens * 0.01` | Multiplication |
| `/` | `output_tokens * (0.03 / 1000)` | Division |
| `//` | `input_tokens // 1000` | Floor division |
| `%` | `input_tokens % 1000` | Modulo |
| `**` | `input_tokens ** 2` | Exponentiation |

## Comparisons

| Operator | Example |
|----------|---------|
| `==` | `output_tokens == 0` |
| `!=` | `output_tokens != 0` |
| `<` | `output_tokens < 1000` |
| `<=` | `output_tokens <= 1000` |
| `>` | `output_tokens > 1000` |
| `>=` | `output_tokens >= 1000` |
| `in` | `"gpt-4" in model` |
| `not in` | `"batch" not in job_type` |

## Boolean

| Operator | Example |
|----------|---------|
| `and` | `tool_calls > 0 and tool_calls <= 10` |
| `or` | `tool_calls == 0 or cache_read_tokens > 0` |

## Ternary

Python-style conditional expression:

```
output_tokens * 0.5 if output_tokens > 1000 else output_tokens * 0.3
```

## Functions

| Function | Description |
|----------|-------------|
| `ceil(x)` | Round up to nearest integer |
| `floor(x)` | Round down to nearest integer |
| `min(a, b)` | Smaller of two values |
| `max(a, b)` | Larger of two values |
| `round(x)` | Round to nearest integer |

## Safety

- Python: AST-based validator with strict node allowlist. No `eval()`, no `exec()`, no attribute access, no imports.
- JavaScript: Recursive descent parser with strict allowlist. No `eval()`, no `Function()` constructor.
- All expressions validated at config load time. Invalid configs never reach the engine.
